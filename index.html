Eyemcast HTML/CSS Style Guide

1 목적

이 문서는 HTML 및 CSS에 대한 형식 및 스타일 규칙을 정의합니다. 협업, 코드 품질을 개선하고 지원 인프라를 활성화하는 것을 목표로 합니다. SCSS 파일을 포함하여 HTML 및 CSS를 사용하는 원시 작업 파일에 적용됩니다.

2 일반

2.1 일반 스타일 규칙

2.1.1 프로토콜

가능한 경우 포함된 리소스에 HTTP 및 HTTPS를 생략합니다.

HTTPS만 접근이 가능한 경우를 제외하고 //(domain)로 시작하십시오.

<!-- 권장하지않음 -->
<script src="https://ajax.googleapis.com/ajax/libs/jquery/3.4.0/jquery.min.js"></script>

<!-- 권장하지않음: uses HTTP -->
<script src="http://ajax.googleapis.com/ajax/libs/jquery/3.4.0/jquery.min.js"></script>

<!-- 권장 -->
<script src="//ajax.googleapis.com/ajax/libs/jquery/3.4.0/jquery.min.js"></script>

/* 권장하지않음 */
@import 'https://fonts.googleapis.com/css?family=Open+Sans';

/* 권장하지않음: uses HTTP */
@import 'http://fonts.googleapis.com/css?family=Open+Sans';

/* 권장 */
@import '//fonts.googleapis.com/css?family=Open+Sans';

2.2 일반 서식 규칙

2.2.1 들여쓰기

한 번에 2칸 들여쓰기.

Don’t 들여쓰기를 위해 탭을 사용하거나 탭과 공백을 혼합하지 마십시오.

<ul>
  <li>Fantastic
  <li>Great
</ul>

.example {
  color: blue;
}

2.2.2 영문자 사용

소문자만 사용하십시오.

모든 코드는 소문자여야 합니다. 이것은 HTML 요소 이름, 속성, 속성 값, CSS 선택기, 속성 및 속성 값(문자열 제외)에 적용됩니다.

단, 색상코드는 대문자로 합니다.

<!-- 권장하지않음 -->
<A HREF="/">Home</A>

<!-- 권장 -->
<img src="google.png" alt="Google">

/* 권장하지않음 */
color: #e5e5e5;

/* 권장 */
color: #E5E5E5;

2.2.3 후행 공백

후행 공백을 제거하십시오.

후행 공백은 불필요하며 diff를 복잡하게 만들 수 있습니다.

<!-- 권장하지않음 -->
<p>What?_

<!-- 권장 -->
<p>Yes please.

2.3 일반 메타 규칙

2.3.1 인코딩

기본적으로 UTF-8 (no BOM)을 사용하세요.

편집기가 바이트 순서 표시 없이 UTF-8을 문자 인코딩으로 사용하는지 확인하십시오.

2.3.2 코멘트

가능한 경우 필요에 따라 코드를 설명합니다.

주석을 사용하여 코드를 설명하십시오. 코드는 무엇을 다루고, 어떤 목적을 수행하며, 각 솔루션이 사용되거나 선호되는 이유는 무엇입니까?

3 HTML

3.1 HTML 스타일 규칙

3.1.1 문서 유형

HTML5를 사용합니다.

가능한 경우 HTML 싱글 태그의 끝을 닫지 마세요 <br>, not <br />.

3.1.2 HTML Validity

가능한 경우 유효한 HTML을 사용하십시오.

W3C HTML 유효성 검사기 와 같은 도구를 사용 하여 테스트합니다.

유효한 HTML을 사용하는 것은 측정 가능한 기준 품질 속성으로 기술 요구 사항 및 제약 조건에 대한 학습에 기여하고 적절한 HTML 사용을 보장합니다.

<!-- 권장하지않음 -->
<title>Test</title>
<article>This is only a test.

<!-- 권장 -->
<!DOCTYPE html>
<meta charset="utf-8">
<title>Test</title>
<article>This is only a test.</article>

3.1.3 의미론

목적에 따라 HTML을 사용하십시오.

생성된 목적에 대해 요소를 사용합니다. 예를 들어, 문단에는, p 요소를, 연결은 a 요소를 사용하십시오.

HTML을 목적에 따라 사용하는 것은 접근성, 재사용 및 코드 효율성을 위해 중요합니다.

<!-- 권장하지않음 -->
<div onclick="goToRecommendations();">All recommendations</div>

<!-- 권장 -->
<a href="recommendations/">All recommendations</a>

3.1.4 멀티미디어 폴백

멀티미디어에 대한 대체 콘텐츠를 제공합니다.

canvas를 통한 이미지, 비디오, 애니메이션 객체와 같은 멀티미디어의 경우, 대체 접근 텍스트를 제공할 수 있습니다.
텍스트(alt)와 비디오 및 오디오 스크립트 및 캡션(사용 가능한 경우)에 사용할 수 있습니다.

시각장애인은 @alt 및 다른 사용자 없이 이미지의 내용을 알려주는 몇 가지 신호를 가지고 있습니다. 비디오나 오디오 콘텐츠가 무엇에 관한 것인지 이해할 방법이 없을 수도 있습니다다.

<!-- 권장하지않음 -->
<img src="spreadsheet.png">

<!-- 권장 -->
<img src="spreadsheet.png" alt="Spreadsheet screenshot.">

3.1.5 파일의 분리

불필요하게 많은 파일을 만들지 마세요.

구조(HTML), 프리젠테이션(Style), 동작(Script)을 엄격하게 분리하고 이 세 가지 간의 상호 작용을 절대 최소로 유지하려고 합니다.

문서안에 inline 형태로 들어가는 Style이나 Script는 아직 실험적인 경우에만 유지하세요.

작업이 마무리되면 .js 또는 .css 파일 안에 묶어서 배포하며, 역할별로 파일이 나뉘는 경우, minify 를 이용해 최종 배포에서는 통합합니다.

<!-- 권장하지않음 -->
<!DOCTYPE html>
<title>HTML sucks</title>
<link rel="stylesheet" href="base.css" media="screen">
<link rel="stylesheet" href="grid.css" media="screen">
<link rel="stylesheet" href="print.css" media="print">
<h1 style="font-size: 1em;">HTML sucks</h1>
<p>I’ve read about this on a few sites but now I’m sure:
  <u>HTML is stupid!!1</u>
<center>I can’t believe there’s no way to control the styling of
  my website without doing everything all over again!</center>

<!-- 권장 -->
<!DOCTYPE html>
<title>My first CSS-only redesign</title>
<link rel="stylesheet" href="default.css">
<h1>My first CSS-only redesign</h1>
<p>I’ve read about this on a few sites but today I’m actually
  doing it: separating concerns and avoiding anything in the HTML of
  my website that is presentational.
<p>It’s awesome!

3.1.6 URL 인코팅 형식 사용 자제

URL 인코딩 형식을 되도록 사용하지 마세요.

근래의 브라우저는 대부분 URL 인코딩을 따로 하지 않아도 정확하게 이해하고 표현합니다. &mdash;, &rdquo;, &#x263a;, 등 더이상 이런 표현을 직접 사용할 필요가 없습니다.

유일한 예외는 HTML을 의미하는 (like <과 &), 그리고 “공백” 문자입니다.

<!-- 권장하지않음 -->
The currency symbol for the Euro is &ldquo;&eur;&rdquo;.

<!-- 권장 -->
The currency symbol for the Euro is “€”.

3.1.7 type 속성은 표기하지 마세요.

스크립트와 스타일에서 type 속성은 더이상 사용되지 않습니다.

<!-- 권장하지않음 -->
<link rel="stylesheet" href="https://www.google.com/css/maia.css"
    type="text/css">

<!-- 권장 -->
<link rel="stylesheet" href="https://www.google.com/css/maia.css">

<!-- 권장하지않음 -->
<script src="https://www.google.com/js/gweb/analytics/autotrack.js"
    type="text/javascript"></script>

<!-- 권장 -->
<script src="https://www.google.com/js/gweb/analytics/autotrack.js"></script>

3.2 HTML 형식 규칙

3.2.1 일반 서식

모든 블록, 목록 또는 테이블 요소에 새 줄을 사용하고 이러한 모든 자식 요소를 들여씁니다.

요소의 스타일 지정과는 별개로(CSS에서는 요소가 display 속성 별로 다른 역할을 맡을 수 있으므로 ) 모든 블록, 목록 또는 테이블 요소를 새 줄에 넣습니다.

또한 블록, 목록 또는 테이블 요소의 자식 요소인 경우 들여쓰기합니다.

(목록 항목 사이의 공백 문제가 발생하면 모든 ’요소를 한 줄 에 넣어도 됩니다. 오류 대신 경고를 표시하는 것이 좋습니다.

<blockquote>
  <p><em>Space</em>, the final frontier.</p>
</blockquote>

<ul>
  <li>Moe
  <li>Larry
  <li>Curly
</ul>

<table>
  <thead>
    <tr>
      <th scope="col">Income
      <th scope="col">Taxes
  <tbody>
    <tr>
      <td>$ 5.00
      <td>$ 4.50
</table>

3.2.2 HTML 줄 바꿈

긴 줄을 끊습니다(선택 사항).

HTML에 대한 열 제한 권장 사항은 없지만 가독성이 크게 향상되면 긴 줄을 줄 바꿈하는 것을 고려할 수 있습니다.

줄 바꿈할 때 각 연속 줄은 원래 줄에서 최소 4칸 더 들여쓰기해야 합니다.

3.2.3 HTML 따옴표

속성 값을 인용할 때 큰따옴표를 사용하십시오.

속성값 ("") rather than single quotation marks ('') around attribute values.

<!-- 권장하지않음 -->
<a class='maia-button maia-button-secondary'>Sign in</a>

<!-- 권장 -->
<a class="maia-button maia-button-secondary">Sign in</a>

4 CSS

4.1 CSS 스타일 규칙

4.1.1 CSS 유효성

가능한 경우 유효한 CSS를 사용하십시오.

W3C CSS validator 와 같은 도구를 사용 하여 테스트합니다.

4.1.2 ID 및 클래스 이름 지정

의미 있는 또는 일반 ID 및 클래스 이름을 사용하십시오.

표현적이거나 비밀스러운 이름 대신 항상 문제의 요소의 목적을 반영하거나 일반적인 경우 ID와 클래스 이름을 사용하십시오.

구체적이고 요소의 목적을 반영하는 이름은 가장 이해하기 쉽고 변경될 가능성이 가장 적기 때문에 선호해야 합니다.

일반 이름은 단순히 형제와 다른 의미가 없거나 특별한 의미가 없는 요소에 대한 대체입니다. 그들은 일반적으로 “helpers.”로 표기합니다.

스크립트 명칭을 동일하게 사용하지 마세요.

/* 권장하지않음: meaningless */
#yee-1901 {}

/* 권장하지않음: presentational */
.button-green {}
.clear {}

/* 권장: specific */
#gallery {}
#login {}
.video {}

/* 권장: generic */
.aux {}
.alt {}

4.1.3 ID 및 클래스 이름 스타일

ID와 클래스 이름은 가능한 한 짧지만 필요한 만큼 사용하십시오.

ID나 클래스가 무엇인지 최대한 간략하게 전달하십시오.

이러한 방식으로 ID와 클래스 이름을 사용하면 수용 가능한 수준의 이해도와 코드 효율성에 기여합니다.

/* 권장하지않음 */
#navigation {}
.atr {}

/* 권장 */
#nav {}
.author {}

4.1.4 Type Selectors

type selectors로 ID 및 클래스 이름을 한정하지 마십시오.

필요한 경우가 아니면(예: 도우미 클래스) ID 또는 클래스와 함께 요소 이름을 사용하지 마십시오.

불필요한 조상 선택자를 피하는 것은 성능상의 이유로 유용합니다. 다음을 참고하세요. CSS성능.

/* 권장하지않음 */
ul#example {}
div.error {}

/* 권장 */
#example {}
.error {}

4.1.5 약식 속성

가능한 경우 약식 속성을 사용하십시오.

CSS는 단 하나의 값만 명시적으로 설정된 경우에도 가능한한 다양한 약식 속성 (예 font)을 제공합니다.

약식 속성을 사용하면 코드 효율성과 이해도에 유용합니다.

/* 권장하지않음 */
border-top-style: none;
font-family: palatino, georgia, serif;
font-size: 100%;
line-height: 1.6;
padding-bottom: 2em;
padding-left: 1em;
padding-right: 1em;
padding-top: 0;

/* 권장 */
border-top: 0;
font: 100%/1.6 palatino, georgia, serif;
padding: 0 1em 2em;

4.1.6 0과 단위

필요한 경우가 아니면 “0” 값 뒤에 단위 지정을 생략합니다.

0 필요한 경우가 아니면 값 뒤에 단위를 사용하지 마십시오.

flex: 0px; /* This flex-basis component requires a unit. */
flex: 1 1 0px; /* Not ambiguous without the unit, but needed in IE11. */
margin: 0;
padding: 0;

4.1.7 선행 0

값에서 선행 “0”을 생략합니다.

font-size: .8em;

4.1.8 16진수 표기법

가능하면 3자의 16진수 표기법을 사용하십시오.

허용하는 색상 값의 경우 3자 16진수 표기법이 더 짧고 간결합니다.

/* 권장하지않음 */
color: #eebbcc;

/* 권장 */
color: #ebc;

4.1.9 접두사

프로젝트별로 고유한 접두사를 사용하세요. (선택사항)

대규모 프로젝트와 다른 프로젝트 또는 외부 사이트에 포함된 코드의 경우 ID 및 클래스 이름에 접두사(네임스페이스로)를 사용합니다. 짧고 고유한 식별자 뒤에 대시를 사용하세요.

네임스페이스를 사용하면 이름 충돌을 방지하고 검색 및 바꾸기 작업과 같은 유지 관리를 더 쉽게 할 수 있습니다.

.adw-help {} /* AdWords */
#maia-note {} /* Maia */

4.1.10 ID 및 클래스 이름 구분 기호

ID와 클래스 이름의 단어는 하이픈으로 구분합니다.

이해와 가독성을 높이기 위해 선택기에서 단어와 약어를 하이픈 이외의 문자(전혀 없는 경우 포함)로 연결하지 마십시오.

/* 권장하지않음: does not separate the words “demo” and “image” */
.demoimage {}

/* 권장하지않음: uses underscore instead of hyphen */
.error_status {}

/* 권장 */
#video-id {}
.ads-sample {}

4.2 CSS 서식 규칙

4.2.1 선언 순서

선언을 알파벳순으로 만듭니다.

기억하고 유지하기 쉬운 방식으로 일관된 코드를 달성하기 위해 선언을 알파벳 순서로 배치하십시오.

정렬을 위해 공급업체별 접두사는 무시합니다. 그러나 특정 CSS 속성에 대한 여러 공급업체별 접두사는 정렬된 상태로 유지해야 합니다(예: -moz 접두사가 -webkit 앞에 옴).

background: fuchsia;
border: 1px solid;
-moz-border-radius: 4px;
-webkit-border-radius: 4px;
border-radius: 4px;
color: black;
text-align: center;
text-indent: 2em;

4.2.2 콘텐츠 들여쓰기 차단

모든 블록 콘텐츠를 들여씁니다.

@media screen, projection {

  html {
    background: #fff;
    color: #444;
  }

}

4.2.3 선언 종료

모든 선언 뒤에 세미콜론을 사용하십시오.

일관성과 확장성을 위해 모든 선언을 세미콜론으로 끝내십시오.

/* 권장하지않음 */
.test {
  display: block;
  height: 100px
}

/* 권장 */
.test {
  display: block;
  height: 100px;
}

4.2.4 속성 이름 종료

속성 이름의 콜론 뒤에 ’을 사용하십시오.

일관성을 위해 항상 속성과 값 사이에 공백을 하나만 사용합니다(속성과 콜론 사이에는 공백 없음).

/* 권장하지않음 */
h3 {
  font-weight:bold;
}

/* 권장 */
h3 {
  font-weight: bold;
}

4.2.5 선언 블록 분리

마지막 선택자와 선언 블록 사이에 공백을 사용하십시오.

마지막 선택자와 선언 블록 을 시작하는 여는 중괄호 사이에는 항상 공백 하나를 사용하십시오 .

여는 중괄호는 주어진 규칙의 마지막 선택자와 같은 줄에 있어야 합니다.

/* 권장하지않음: missing space */
#video{
  margin-top: 1em;
}

/* 권장하지않음: unnecessary line break */
#video
{
  margin-top: 1em;
}

/* 권장 */
#video {
  margin-top: 1em;
}

4.2.6 선택자와 선언 분리

새 줄로 선택자와 선언을 구분합니다.

각 선택자와 선언에 대해 항상 새 줄을 시작합니다.

/* 권장하지않음 */
a:focus, a:active {
  position: relative; top: 1px;
}

/* 권장 */
h1,
h2,
h3 {
  font-weight: normal;
  line-height: 1.2;
}

4.2.7 규칙 분리

새 줄로 규칙을 구분합니다.

규칙 사이에는 항상 빈 줄(두 줄 바꿈)을 넣으십시오.

html {
  background: #fff;
}

body {
  margin: auto;
  width: 50%;
}

4.2.8 CSS 따옴표

속성 선택기 및 속성 값에 큰따옴표("")대신 작은따옴표('')를 씁니다.

URL값에는 따옴표를 사용하지 마세요. (url()).

예외: @charset을 사용하는 경우에는 큰 따옴표를 사용하세요. 작은 따옴표는 허용되지 않습니다..

/* 권장하지않음 */
@import url("https://www.google.com/css/maia.css");

html {
  font-family: "open sans", arial, sans-serif;
}

/* 권장 */
@import url(https://www.google.com/css/maia.css);

html {
  font-family: 'open sans', arial, sans-serif;
}
